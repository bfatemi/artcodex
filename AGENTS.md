# AGENTS.md

This repository contains `artcodex`, a community-maintained R client for the
local Codex app-server. Keep the package generic, deterministic, and suitable
for public distribution. Do not add organization-specific workflows, private
paths, credentials, or assumptions about a particular Codex account.

## Product boundary

- `artcodex` owns app-server process management, transport, protocol events,
  thread and turn control, and serializable R results.
- The Codex CLI owns authentication, model access, sandbox execution, MCP
  servers, plugins, skills, and tool behavior.
- Higher-level packages own domain workflows. Keep those out of this package.
- The package is an R client, not a replacement for the Codex CLI or terminal
  interface and not an official OpenAI SDK.

## Source map

- `R/client.R`: executable discovery and app-server lifecycle.
- `R/json-rpc.R`: line transport, message routing, response caching, events,
  and server-initiated request handling.
- `R/thread.R`: thread lifecycle and the public `codex_run()` workflow.
- `R/events.R`: final-response extraction and result normalization.
- `R/schema.R`: schema generation for the installed CLI.
- `R/conditions.R`: public success checks and condition helpers.
- `R/utils.R`: validation, JSON, timing, and list helpers.
- `tests/testthat/fixtures/`: deterministic fake app-server behavior.

## Protocol invariants

- Treat standard input/output as a newline-delimited message stream.
- Never discard a response because it arrived before another requested ID.
- Drain buffered output before reporting that the process exited.
- Use one monotonic deadline across request and completion waits.
- Match turn events by both thread ID and turn ID.
- Treat only `completed` turns as successful.
- Interrupt a known running turn after timeout or user interruption.
- Default all unhandled server requests to a conservative response.
- Preserve raw messages and normalized events for diagnosis.
- Validate behavior against schemas generated by the currently supported CLI;
  do not infer protocol shapes from UI behavior.

## Public API principles

- Keep the default path simple: `codex_run("prompt")` owns cleanup.
- Keep pipeline control explicit through client and thread objects.
- Return serializable lists by default; do not place process environments in
  `artcodex_result`.
- Add new top-level result fields without changing the meaning of existing
  fields.
- Validate user input before starting a process or turn.
- Keep defaults read-only and non-interactive.
- Document every export with roxygen2 and include runnable or `\dontrun{}`
  examples as appropriate.

## Development commands

Run commands from the repository root. On Windows, clear unsupported locale
overrides if they are inherited from the shell.

```r
devtools::document()
testthat::test_local()
lintr::lint_package()
covr::package_coverage()
rcmdcheck::rcmdcheck(args = c("--no-manual", "--as-cran"), error_on = "warning")
```

Set `ARTCODEX_LIVE_TESTS=1` only for an authenticated live smoke test. Ordinary
tests must not require Codex, network access, Node.js, credentials, or a user
configuration directory.

## Change checklist

1. Update implementation and focused deterministic tests together.
2. Regenerate `NAMESPACE` and `man/` with roxygen2.
3. Update `NEWS.md` for user-visible behavior.
4. Run tests, lint, coverage, and `R CMD check --as-cran`.
5. When protocol behavior changes, generate the current schema and run a live
   initialize/turn smoke test when credentials are available.
6. Inspect the built source tarball for secrets, private paths, and unintended
   generated files before publishing.
