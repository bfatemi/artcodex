# Security Policy

## Supported versions

Security fixes are made on the latest released version and the default branch.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability or include secrets,
authentication material, private prompts, or sensitive protocol transcripts in
public reports. Use GitHub's private vulnerability reporting form on the
repository's **Security** tab.

Include the affected version, operating system, Codex CLI version, reproduction
steps, impact, and any suggested mitigation. The maintainer will acknowledge a
report, investigate it, and coordinate disclosure and a release when needed.

## Scope

Reports about process management, protocol parsing, approval handling, sandbox
configuration, unintended data retention, or credential exposure in `artcodex`
are in scope. Vulnerabilities in Codex itself or in third-party tools should be
reported to their respective maintainers.
